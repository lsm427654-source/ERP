# AI 에이전트 협업 튜토리얼: SAP FTA 시뮬레이터 구축 과정

본 문서는 **현대오토에버 SAP ERP 직무** 지원을 위한 포트폴리오 프로젝트인 **'SAP 연계 FTA 원산지 판정 시뮬레이터'**를 인공지능 에이전트와 함께 처음부터 끝까지 구축한 과정을 기록한 튜토리얼입니다.
아이디어 구상부터 구현, 문서화까지 사용된 **프롬프트(명령어)**와 에이전트의 **수행 결과**를 중심으로 정리했습니다.

---

## 1. 프로젝트 착수 및 환경 설정 (Repository Setup)

가장 먼저 프로젝트를 진행할 작업 공간(GitHub Repository)을 설정했습니다.

### 🗣️ User Prompt
> "https://github.com/lsm427654-source/ERP.git 여기랑 연결해줘"

### 🤖 AI Action
1.  **Repository Clone**: 지정된 깃허브 주소로부터 프로젝트를 로컬(`c:\Users\WIN\Desktop\ERP`)로 복제했습니다.
2.  **초기화**: 기존에 잘못 연결된 폴더가 있다면 정리하고, 깨끗한 상태에서 시작하도록 환경을 구성했습니다.

---

## 2. 아이디어 구체화 및 요구사항 정의 (Idea & Requirements)

단순한 구현이 아니라, **'현대오토에버 SAP ERP'**라는 명확한 타겟을 설정하고, 이를 위한 맞춤형 프로젝트를 제안하고 구체화했습니다.

### 🗣️ User Prompt
> "나는 현대오토에버 SAP ERP 직무에 지원할 예정이야. 내 포트폴리오용으로 [SAP 연계 FTA 원산지 판정 시뮬레이터]를 만들고 싶어.
> 기술 스택은 [Python, Streamlit, SQLite]를 사용해서 로컬 MVP 버전으로 만들어줘.
>
> [요구사항 상세]
> 1. DB 설계: SAP 테이블 모사 (ZMM_MATERIAL, ZPP_BOM 등) 및 더미 데이터(전기차 배터리).
> 2. 비즈니스 로직: CTSH(세번변경기준) 알고리즘 구현.
> 3. UI 구성: 좌측 마스터 데이터, 우측 판정 시뮬레이터."

### 🤖 AI Action
1.  **구현 계획(Implementation Plan) 수립**: 사용자의 요청을 분석하여 `app.py` 구조와 데이터베이스 스키마를 기획했습니다.
2.  **데이터 모델링**: SAP의 실제 테이블 구조를 본뜬 `ZMM_MATERIAL`(자재), `ZPP_BOM`(BOM) 테이블 설계안을 제시했습니다.
3.  **시나리오 설정**: 현대오토에버의 산업 도메인(자동차 부품)에 맞춰 '전기차 배터리 팩'과 중국산 '리튬 셀'이라는 구체적인 비즈니스 시나리오를 설정했습니다.

---

## 3. 코드 구현 및 검증 (Implementation & Verification)

계획된 내용을 실제 동작하는 코드로 작성했습니다.

### 🗣️ User Prompt
> "코드를 주석과 함께 아주 상세하게 짜줘."

### 🤖 AI Action
1.  **`app.py` 작성**:
    -   **DB Init**: SQLite DB 연결 및 테이블 생성, 더미 데이터 적재 함수 구현.
    -   **Logic**: BOM을 재귀적으로 탐색(`explode_bom`)하고, HS Code를 비교하여 원산지를 판정(`determine_origin_ctsh`)하는 핵심 로직 구현.
    -   **UI**: Streamlit을 활용해 2단 레이아웃(Master Data vs Simulator) 구현.
2.  **`requirements.txt` 작성**: `streamlit`, `pandas` 등 필수 라이브러리 명시.
3.  **검증(Verification)**: `test_app.py` 스크립트를 생성하여 UI 실행 전 로직의 정상 작동 여부(Foreign 판정 로직 등)를 미리 테스트했습니다.
